Review the current project, then create or update the README.md following the standard format defined in the global CLAUDE.md (~/.claude/CLAUDE.md).

Steps:
1. Read the existing README.md if one exists
2. Explore the repo structure enough to understand: what the project does, what stack it uses, how to run it, and how to deploy it (if applicable)
3. Read the global README format standard from ~/.claude/CLAUDE.md
4. Write or rewrite the README following that standard — including all sections that apply, omitting sections that don't
5. Use the rag-auditor README (https://github.com/amitgambhir/rag-auditor) as the tone and style reference

Do not invent content. If information isn't derivable from the codebase, leave a clearly marked placeholder like `[add your live URL here]` rather than guessing.
